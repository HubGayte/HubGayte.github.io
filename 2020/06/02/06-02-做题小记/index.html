下面的随机数没有使用种子，而是直接生成的，而且使用 hash_equals 避免了时序攻击和 php 弱类型，所以无法利用

用来获取当前页面的名字，正常情况下和 __FILE__ 意义相同，但是在 1.php/2.php 这种情况下就会获取 1.php/2.php，而浏览器和服务器对这种 url 的解析结果都是 1.php，这样我们就可以移花接木，读取到 config.php 的源码。

Code

这个正则的意思是说匹配的是 config.php 后面加任意个数的 /

因此只要结尾不是/config.php/就能绕过这个正则，但同时要求我们的basename取出来后是config.php

basename() 函数返回路径中的文件名部分。它会忽略不可显字符[\x80-\xff]范围内的字符

可以通过config.php后添加[\x80-\xff]范围内的字符来绕过正则匹配，同时使得basename取值仍为config.php
