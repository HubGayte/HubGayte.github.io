基础知识

1.任何具有一定结构的数据，只要经过了某些处理而把自身结构改变，则可能会产生漏洞。

2.PHP 在反序列化时，底层代码是以 ; 作为字段的分隔，以 } 作为结尾(字符串除外)，并且是根据长度判断内容的；PHP 在反序列化时，对类中不存在的属性也会进行反序列化。

3.反序列化的对象逃逸问题一般分为两种

①关键词数增加

例如： [0CTF 2016]piapiapia

php

where->hacker，这样字母数由五个增加到6个

因此我们可以将where后拼接上我们想要的语句，并计算好单词替换次数，通过闭合}即可达到我们字符逃逸的效果

②关键词数减少

例如：[安洵杯 2019]easy_serialize_php

php

题目中直接过滤掉一些关键词

这种可以通过键逃逸和值逃逸来绕过

payload1：键逃逸：我们直接构造会被过滤的键，这样值的一部分充当键，剩下的一部分作为单独的键值对

payload2：值逃逸：需要构造两个连续的键值对，由第一个的值覆盖第二个的键，这样第二个值就逃逸出去，单独作为一个键值对

4.数组绕过正则即相关

md5(Array()) = null、sha1(Array()) = null、ereg(pattern,Array()) =null、preg_match(pattern,Array()) = false、strcmp(Array(), “abc”) =null、strpos(Array(),“abc”) = null、strlen(Array()) = null
